1. System Architecture

Pulse AI is a three-tier, locally-hosted stack tunnelled to a public GitHub Pages front-end. No user data or credentials are stored server-side at any point.

Why ngrok instead of Streamlit Cloud?

Streamlit Cloud injects X-Frame-Options: SAMEORIGIN at the infrastructure level, which prevents the app from being embedded in an iframe on a different origin. Running Streamlit locally through ngrok gives full control over response headers — no iframe restrictions apply.

Request flow for a health report

  1. User selects an account in the sidebar and clicks Generate Health Report.
  2. Streamlit sets st.session_state.pending_generate and calls st.rerun() — the browser paints the loading state before the blocking network request starts.
  3. On the next rerun, Streamlit posts POST {LANGFLOW_URL} with input_value = account_id (e.g. ACC-003).
  4. Langflow's RAG pipeline retrieves all vectors tagged with that account ID, synthesises the report, and returns structured Markdown.
  5. The response is parsed by parse_sections(), classified by classify_section(), and rendered through the section-specific renderer pipeline.

2. Credential & Settings Flow

Credentials travel from the user's browser to Streamlit via URL query parameters. They are never committed to source code and never sent to any third-party service.

1
start.bat writes to localStorage

After tunnels are live, the batch script generates a temporary HTML page that writes { streamlitUrl, url, portfolioUrl, key } to localStorage["pulseai.langflow.settings.v1"] and then redirects to the doc site.

2
site.js reads localStorage on DOMContentLoaded

initIframe() in assets/js/site.js calls readSettings(). If settings are present it calls buildStreamlitUrl(lf) which appends ?lf_url=…&lf_key=…&lf_portfolio=… to the ngrok Streamlit base URL, then sets iframe.src.

3
Streamlit reads query params on each request

_get_credential(qp_key, env_key) in pulse_app.py checks st.query_params first, then os.environ, then falls back to an empty string (shell-only mode).

4
Langflow calls are server-side only

All requests.post() calls happen inside Streamlit's Python process. The API key is never sent to the browser or written to a log file.

5
Manual override via Connect modal

The Connect button in the doc-site header opens a settings modal. On Save, writeSettings() persists to localStorage and reloadIframe() rebuilds and resets the iframe src.

5. Report Rendering Pipeline

Raw Markdown returned by Langflow passes through a deterministic five-stage pipeline before reaching the browser.

StageFunctionDescription
1 — Parseparse_sections(text)Splits on ## / ### headers and standalone bold lines (**Title**:). Returns a list of {title, body[]} dicts.
2 — Classifyclassify_section(title)Regex-based category assignment: risk | usage | support | financial | action | overview | general.
3 — Meta lookupSECTION_META[cat]Fetches colour, background, icon SVG path, and display label for the section's header chip.
4 — Render bodyrender_section_body(cat, body_md)Routes to the category-specific renderer (see below). Falls back to md_to_html() for unrecognised categories.
5 — Card wrapcard_wrap(content, border_color)Wraps HTML in a styled <div> with left accent border, border-radius, and optional height:100% for equal-height columns.

Section-specific renderers

CategoryRendererOutput components
overviewrender_overview()Key–value rows for Overall Status, Primary Driver, Risk Level; narrative paragraphs for everything else.
usagerender_usage()Stat tiles (number + trend arrow) for DAU / NPS / adoption metrics; positive and warning bullet lists; adoption insight callout.
supportrender_support()Ticket cards with severity and status badges for each TKT-NNN reference; summary bullet list.
financialrender_financial()Key–value grid for contract fields (ARR, Renewal Date, Plan Tier); expansion signal callout block.
riskrender_risk()Score ring SVG (0–100), risk level badge, numbered risk-factor list, confidence meter (3-dot indicator).
actionrender_actions()Numbered priority action cards, each with a What and Why field and a colour-coded index dot (coral → amber → teal).

Text analysis helpers

  • detect_status(text) — regex scans for Overall Status: Critical/At Risk/Healthy then keyword fallback → returns one of Healthy | At Risk | Critical | Reviewed.
  • extract_score(text) — captures integer 0–100 after health score: pattern; returns None if absent.
  • extract_confidence(text) — extracts High | Medium | Low from Confidence: … line.

6. Portfolio Overview Engine

The portfolio panel loads automatically on first session start (portfolio_cache is None and portfolio_error is None) and can be manually refreshed via the ↻ Refresh button above the expander.

Data flow

  1. fetch_portfolio() posts to the Portfolio Aggregator flow.
  2. The response contains two output nodes: the table node (identified by display_name == "Portfolio Aggregator") and the narrative node.
  3. parse_portfolio_table(text) splits the pipe-delimited table, skipping the header row, and builds { counts, top5, all }.
  4. Results are stored in st.session_state.portfolio_cache. Errors are stored in st.session_state.portfolio_error.
  5. The donut SVG chart and Top-5 attention table are rendered inside equal-height columns using card_wrap(..., _full_h=True).

Session state keys

KeyTypeDescription
report_cachedict[str, dict]Per-account report cache. Keyed by account name. Each value: {explanation, generated_at, acc_id}.
portfolio_cachedict | NonePortfolio data: {chart: {counts, top5, all}, narrative: str}. None triggers auto-load.

Manual startup (without start.bat)

# 1. Install dependencies (once only)
pip install -r requirements.txt

# 2. Start Streamlit with credentials as env vars
LANGFLOW_URL=https://aaa.ngrok-free.app/api/v1/run/<account-flow-id> \
LANGFLOW_PORTFOLIO_URL=https://aaa.ngrok-free.app/api/v1/run/<portfolio-flow-id> \
LANGFLOW_API_KEY=sk-... \
python -m streamlit run app/pulse_app.py

# 3. Serve the doc site locally (separate terminal)
python -m http.server 5500

Streamlit configuration (.streamlit/config.toml)

[server]
enableCORS  = false
headless    = true

enableCORS = false is required to allow the ngrok-tunnelled Streamlit app to be loaded inside the GitHub Pages iframe from a different origin.

8. Security Model

ControlImplementation
No credentials in sourceAll secrets flow via query params or env vars — nothing hard-coded except flow IDs (non-secret).
No credentials in git history.gitignore excludes .env, *.bat variants, and ngrok token files.
API key storageStored in browser localStorage under a versioned key. Never written to disk or logged server-side.
LLM isolationLLM API key is held by Langflow Desktop only. Streamlit never receives or forwards it.
ngrok token rotationToken in start.bat — rotate before sharing the repository publicly.
iframe CORSenableCORS = false in Streamlit config + ngrok HTTPS tunnel. No wildcard CORS headers on the doc site.
Open sourceAll application code is public. No obfuscation.
Public demo note
The ngrok free tier generates a new public URL on each tunnel restart. If sharing the demo with stakeholders, either keep start.bat running or upgrade to a paid ngrok plan with a fixed reserved domain.
Account data is synthetic
The 20 accounts (ACC-001 → ACC-020) reference fictional company names. All health signals, ticket IDs, and contract values are generated by the RAG knowledge base and are not derived from real customer data.
